분석 배경 및 방법론

본 보고서는 시장분석가 에이전트(시장 규모·경쟁사·수익 모델·마케팅 채널 담당)와 게임 시스템 설계 에이전트(수치 공식·리텐션 설계·콘텐츠 깊이·스코프 확정 담당)가 독립적으로 분석한 결과를 종합하여 작성하였다. v1.0 GDD에서 발견된 오류(KPI 수학적 불일치, Magic Research 잘못된 벤치마크 적용)를 수정하고 현실 기반의 설계를 재수립하는 것이 분석의 출발점이다.

Section 02 시장 환경 분석

방치형(Idle) 게임 시장 현황

방치형 RPG는 2020년 이후 모바일 게임 시장에서 가장 빠르게 성장한 세부 장르 중 하나다. 핵심 특성은 낮은 동시접속 요구, 오프라인 수익 구조로 인한 DAU 대비 높은 LTV, 1인 개발 가능한 기술 복잡도다.

장르 강점
DAU 100명부터 수익 발생 가능. 유저가 앱을 켜지 않아도 BK가 쌓이는 구조가 D30 이후 리텐션을 자연적으로 유지. 재화 구조가 단순해 IAP 설계가 직관적.
경쟁 강도
한국어 방치형 RPG 시장은 포화 직전이나, 무속/신당 소재는 경쟁작 사실상 0. 일본 신사·중국 선인 소재는 다수 존재하나 한국 무속 소재는 공백.
K-콘텐츠 프리미엄
무당·굿판 소재는 넷플릭스 한국 드라마(스물다섯, 신과 함께 등)로 글로벌 인지도가 형성됨. 영어권 유저의 exotic appeal 유효.

Section 04 수익 모델 설계

모델 선택 근거

검토한 4개 모델(프리미엄 $1~5, 광고 위주 F2P, IAP 위주 F2P, 하이브리드 F2P) 중 IAP 박리다매 주력 + 보상형 광고 보조 하이브리드가 1인 개발자 기준 최적이다. 이유는 세 가지: ① 손익분기 DAU가 100명 수준으로 낮아 초기 리스크 최소화, ② 월정액 패스가 LTV를 장기간 안정적으로 유지, ③ 보상형 광고로 비결제 유저도 리텐션 유지.

배제된 모델
광고 위주 모델(배너+전면 과다)은 r/incremental_games, 디시 방치형 갤 모두 적대적 반응이 확인됨. "광고 게임"이라는 낙인은 ASO 별점 1점 테러로 직결. 배너 광고는 완전히 제거, 전면 광고는 자연 휴지점(시대 전환·대굿 완료)에만 제한.

Section 05 핵심 수치 시스템

게임 밸런스의 핵심은 "항상 구매할 것이 있고, 항상 다음 목표가 보이는" 상태를 유지하는 것이다. 아래 공식은 Unity C# 코드에 직접 사용 가능하도록 설계되었다.

① 복채(BK) 생산 공식

// 전체 복채/s — 모든 신당 시설 합산 BK_per_second = Σ ( base × LevelMult(lv) × SpiritMult × RegionMult × EraMult ) // 레벨 배율 — 지수 압축으로 무한 성장 방지 LevelMult(lv) = lv ^ 1.5 × 1.2 ^ (lv / 25) // 업그레이드 비용 — 구간별 압축 계수 적용 Cost(lv) = baseCost × 1.15 ^ lv × Compress(lv) // lv < 25 → Compress = 1.00 // lv < 50 → Compress = 0.85 // lv < 100 → Compress = 0.70

설계 의도: lv^1.5 는 초반 성장을 빠르게 체감시키고, 1.2^(lv/25) 는 장기 플레이어에게 지속적 성장 만족감을 제공한다. Compress 계수는 고레벨 업그레이드가 체감적으로 "비싸지지 않도록" 완화하여 과금 압박을 줄인다.

② 오프라인 수익 공식

offline_BK = BK_per_second × min(elapsed, cap) × efficiency × shamanBonus // efficiency 단계 기본: 0.25 (25%) 자동화 제단 구매: 0.50 (50%) 야광귀 신령 획득: 0.60 (60%) 프리미엄 패스: 0.75 (75%) // cap (오프라인 시간 상한) 기본: 14,400s ( 4시간) 수호 신령: 28,800s ( 8시간) 3회 전생+: 43,200s (12시간) // shamanBonus — 무당 레벨마다 +0.2% shamanBonus = 1 + shamanLevel × 0.002

설계 의도: 기본 효율 25%는 "오프라인 동안 온라인의 1/4만 번다"는 직관적 이해를 제공한다. 프리미엄 패스가 75%로 올라가는 구조는 패스 구매의 핵심 가치 제안이 된다. 오프라인 시간 상한은 푸시 알림과 연동되어 매일 접속하는 이유를 만든다.

③ 신뢰도(TR) 시스템

// 신령 tier별 강림 요구 신뢰도 TR_required(tier) = 100 × 2.8 ^ (tier - 1) // tier 1→8 실제값 tier 1: 100 (~10분) tier 2: 280 (~30분) tier 3: 784 (~2시간) tier 4: 2,195 (~6시간) tier 5: 6,147 (~15시간) tier 6: 17,211 (~30시간) tier 7: 48,191 (~60시간) tier 8: 134,934 (100시간+) // TR 획득 TR_per_sec = (BK_per_sec × 0.01) + 굿_보너스 // 굿 보너스: 소굿 +50 / 중굿 +200 / 대굿 +1,000

④ 프레스티지(전생) 공식

// SK(신기) 획득량 SK_gain = floor( sqrt(totalBK / 1,000,000) ) × eraMult // 시대별 eraMult 현대(1회차): × 1.0 조선(2회차): × 1.3 (+30% 굿 효율) 고려(3회차): × 1.7 (-50% 지역 비용) 고조선(4회차): × 2.2 (+4시간 오프라인) 신계(5회차): × 3.0 (신계 탐험 해금) // SK 누적 영구 복채 배율 SK 0~500: 각 +1% (최대 +500%) SK 500~2K: 각 +0.5% SK 2K~10K: 각 +0.1% SK 10K+: BK배율 고정 → 신계 전용 화폐로 전환

설계 의도: 단순 배율 리셋이 아닌 "시대가 바뀌면 규칙이 바뀐다"는 구조로, 5번의 전생이 각각 다른 게임처럼 느껴지게 한다. 이는 D30 이후 장기 리텐션의 핵심 기제다.

Section 06 리텐션·콘텐츠 설계

콘텐츠 깊이 분석

v1.0 문제: 30~50시간으로 한계
GDD v1.0의 1레이어 구조(신당 업그레이드만)는 최대 30~50시간 플레이타임에 그친다. 방치형 RPG의 D30+ 리텐션을 위해선 최소 80~120시간 이상의 콘텐츠 깊이가 필요하다. 이를 해결하기 위해 5레이어 시스템으로 재설계했다.
레이어해금 타이밍핵심 시스템추가 플레이타임이탈 방어
L1 — 신당 운영즉시신당 업그레이드, 복채 생산, 굿 수행~15~20h삼신할미 3분 자동 강림
L2 — 무당 고용45분~1시간무당 채용, 스킬 트리 3갈래+10~15h스킬 트리 빌드 고민
L3 — 지역 확장6~8시간3지역 해금, 지역별 신령·수익+20~25h지도 UI "와" 모멘트
L4 — 제자 시스템15~20시간제자 5명, 파견·귀환 타이머+15~20h자동화 확장감
L5 — 신계 탐험30~40시간신계 파견, 특수 재화+20~30h전생 후 목표 유지
합계 (5레이어 × 프레스티지 5회)80~120hD30 5% 잔존 달성 가능
